Represents a Channel using window postMessage mechanism.

Using this consists of four steps:

  1. First, we need to identify the initiating window and the initiatee window.
  2. The initiator window identifies the initiatee window to send message to. It opens the channel using the open method.
  3. The initiatee window needs to know the origin of the initiating window, through some side channel or some preagreed value. It opens the channel using the listen method.
  4. Both the open and listen methods returns a Promise that will be resolved with the channel when the connection is open.

To close the channel, just dispose it.

For example, the initiating window can do:

import PostMessageChannel './ui/post-message-channel';

PostMessagChannel
    .open(window, iframeEl.contentWindow)
    .then((channel: PostMessageChannel) => {
      channel.post({ 'message': 'blah' });
    });

The other window does:

import PostMessageChannel './ui/post-message-channel';

PostMessageChannel
    .listen(window, 'http://expected.origin')
    .then((channel: PostMessageChannel) => {
      return channel.waitForMessage(() => true)
    })
    .then((data: gs.IJson) => {
      console.log(data.message);  // 'blah'
    });

Waits for a message from the destination window.

This method takes in a testFn which determines if the message received is the message that the caller is waiting for. Once the caller has determined that the message is the one it wants, this method will resolve the promise it returns.
